# Contributing to Yaademan

Thank you for helping keep the memory of victims alive.  
Yaademan is a community-driven memorial, and **everyone is welcome to contribute**, whether you are new to GitHub or an experienced developer.

---

## Who Can Contribute?

### ๐ฑ Beginners
You **do not need to be a developer** or Git expert.
If you know basic information about a person and want to help preserve their memory, your contribution is valuable.

### ๐ง Experienced Contributors
If you are comfortable with Git, JSON, or data validation, your help in improving accuracy, structure, and completeness is greatly appreciated.

---

## How to Add a Memorial (Simple Version โ for Beginners)

1. Fork this repository (GitHub will guide you)
2. Create a new JSON file inside `data/people/`
3. Use an existing file as a template
4. Fill in **only the information you are confident about**
5. Submit a Pull Request

> It is completely okay if some fields are missing.

---

## How to Add a Memorial (Advanced Version)

1. Fork and clone the repository
2. Create a new JSON file in `data/people/`
3. Follow the schema in `data/schema/person.schema.json`
4. Use a clear and consistent `person-id`
5. (Optional) Add photos to: images/people/[person-id]/
6. Submit a Pull Request with a clear description

---

## Data Guidelines (Important)

- **Not all information can be fully verified** โ this is understood
- Add only what is known or reasonably believed to be correct
- If you have sources, include them when possible (links, references, notes)
- It is better to submit partial information than none at all
- Be respectful in language and descriptions
- Do not speculate or add unconfirmed details as facts

---

## Photos & Images

- Photos should be respectful memorial images
- Avoid graphic or violent content
- If unsure about an image, leave it out โ it can be added later

---

## Review Process

- All submissions are reviewed before publication
- Maintainers may:
- Request changes
- Edit for clarity or consistency
- Merge partial information and complete it later
- Revisions and corrections are welcome at any time

---

- Open an Issue with information you have
- Submit an incomplete profile
- Ask questions in a Pull Request comment

Every contribution matters.
